## 2024-07-20 - Credit Card PII Leakage and CVV Exposure
**Vulnerability:** The Checkout review page (`Review.js`) displayed the customer's full credit card number in cleartext. Additionally, the CVV input field in the payment form (`PaymentForm.js`) used a standard cleartext text field without hiding input or enforcing length and digit formats.
**Learning:** These practices directly violate PCI-DSS compliance requirements and expose highly sensitive Personally Identifiable Information (PII) to shouldersurfing, local device exposure, or unintended logs. They existed due to missing input masking on display components and lack of security-focused UI element attributes (`type="password"`).
**Prevention:** Always mask sensitive card details on review or profile displays so that only the last 4 digits are visible. Enforce secure input masking for authentication or authentication-adjacent codes (such as CVV) by setting `type="password"`, limiting character length, and restricting character patterns to numeric input only.
