## 2026-07-24 - PCI-DSS Compliance & Display Masking of Sensitive Payment Data
**Vulnerability:** The Checkout flow was displaying the raw, unmasked credit card number on the final order review screen (`Review` component) and the CVV field in the payment form was a plain-text input (`type="text"` instead of `type="password"`) with no digit limits or format controls. This violated PCI-DSS security guidelines and leaked sensitive cardholder PII.
**Learning:** E-commerce features often require payment details for mock checkout processes, leading developers to implement standard forms without full security enforcement. If raw card numbers and CVVs are handled as regular text inputs and displayed directly, they can easily leak to onlookers, screen captures, or log files.
**Prevention:** Always mask credit card numbers on any UI presentation component so that only the last 4 digits are visible. Ensure CVV inputs are set to `type="password"` to obscure the input, bind them properly as controlled React components, and enforce strict maxLength and pattern controls (e.g., maximum of 4 numeric-only digits) to limit input values.
